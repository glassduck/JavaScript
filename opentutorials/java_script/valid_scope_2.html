var 은 function scope이라서 function 내에서 지역변수로 활용되고 function 바깥에서는 전역변수로 활용된다. 그리고 잘못 쓰면 전역변수가 되어서 기존의 값이 바뀌어 버린다. 근데 이건 다 그런거 아닌가.... 그러다 보니 선언을 하지 못하면 아예 사용할 수 없게 let과 const가 등장함. let은 선언 후 재할당 가능 const는 불가능. 둘다 재선언은 불가.... 따라서, 무조건 const 활용하는 것이 좋다고 한다. 그래야 수많은 코드 사이에서 나도 모르는 새 변수 끼리 충돌하고 꼬이는 걸 방지 가능. var은 for문이나 if에서는 지역변수로 활용되지 않으니 그것도 주의!
